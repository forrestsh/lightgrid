# LIG-36 · V2.1 自动验收与 `/v2.1` 部署

## 用户故事

作为 Alpha 评审，我希望自动验证 V2.1 的空间合同、动态拓扑、连续性与部署入口，以便确认演示不是依赖一次人工操作或在线模型的原型。

## 验收标准

- 自动测试覆盖设计文档 AC-V21-01～13：断桥替代路、增量修复、矿城重规划、可见/可达差异、GraphPhase 恢复、身体路线、忠实检索、价值上限、事实边界、无模型主线、离线安全、存档迁移和三次放手。
- 测试同时验证 FCC 偶宇称、连续邻接、SceneManifest 和本地脚本加载顺序。
- 页面不加载远程脚本、字体或在线模型。
- `vercel.json` 同时把 `/v2.1` 与 `/v2.1/` 路由到 `/v2.1/index.html`。
- 旧 `/v2`、首页和其他演示路由保持不变。

## 验证命令

```text
node --test tests/v2-core.test.js tests/v2.1-core.test.js
```

本版本为纯静态 HTML/CSS/JavaScript，不需要构建命令或运行时环境变量。
