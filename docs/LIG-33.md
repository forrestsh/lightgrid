# LIG-33 · 三世界动态拓扑与技能重规划

## 用户故事

作为玩家，我希望桥梁、矿城故障和漂移花园会改变可达路线，以便澄学习的是在立体空间中可重规划的方法，而不是固定坐标脚本。

## 验收标准

- 断桥状态从 broken → temporary → stable 时只标记受影响路线，并在修复后开放主桥捷径。
- 矿城自主考试把故障从 NODE-17/+18 改为 NODE-42/−11，并动态封闭维修螺旋；澄改走货运螺旋但保留四个诊断步骤。
- 漂移花园至少有 phase_a、phase_b、phase_c，分别声明启用路线、FlowField、预告时间与安全锚点。
- 身体能力参与路线筛选：浮游囊使用风门上升流，抓附足可使用岛面抓附链，生态感知器使用相位对齐的根桥。
- 每次选路保存 routeId、phaseId 与重规划原因，供记忆与调试界面使用。
