# LIG-28 · 多次放手、回归摘要与玩家数据控制

## 用户故事

作为玩家，我希望定义离开的边界，让澄独立生活一段有限时间，并在回来时理解它关注什么、做了什么和留下什么承诺；同时我能导出或彻底清除角色数据。

## 验收标准

- 玩家可选择短观察、一段日程或前往另一世界三种离开边界。
- 放手前显示当前意图、安全限制和不可逆行动禁令，不提供逐步遥控。
- 每次回归摘要包含事实、理由、行动、结果、未决事项和关键事件引用。
- 连续三次放手均保留同一角色、技能、关系、作品和记忆；`releaseCount` 持续增长。
- 运行中的每次有效状态变化自动保存到浏览器本地，刷新后恢复。
- 玩家可下载可移植 JSON 存档，也可经二次确认彻底清除本地角色数据。
